Worklog: A simple Linux time-tracking program

Mon, 11 Oct 2010

Posted In: linux

﻿A while back, I finally found a perfect little time-tracking program for me - it has all the features I want and none of what I don't want. Worklog is a Linux program that uses an ncurses interface in the terminal to track your time spent on various projects.

A description from the author of the program:

﻿﻿﻿"Worklog is a program that helps you keep track of your time. I wrote this program because I have several projects that require me to keep a timesheet. I sometimes spend 5 minutes here and there doing miscellaneous administrative tasks. Rather than invoke an editor on a timesheet file for every little segment of my time spent doing these tasks, I wanted to leave a clock running and indicate with one or two keystrokes the onset or change of any given task. Worklog is a simple ncurses based program to do just that."

Worklog can be (kind of) found at:

  • [http://www.truxton.com/~trux/software/][] - the author's home page
  • http://packages.ubuntu.com/search?keywords=worklog - Ubuntu package page, choose your version of Ubuntu (currently Maverick) for a description of the package

In Ubuntu, Worklog can be installed with

sudo apt-get install worklog

It's an older utility that doesn't look to have been updated in the last 4 1/2 years at least, but it gets the job done. When you install it, you can read more about it with

man worklog

More Instructions

Even with the man page, there's not much documentation out there. So I'd like to provide a quick overview and a couple of tips for using Worklog.

Before you start using Worklog, you need to create a "projects" file. Use a text editor to create a text file called "projects" and save it in the current directory. The Worklog man page contains an example of what could go in this file (lines starting with # are comments and do not affect Worklog):

# Worklog project file
# note that projects appear in Worklog in REVERSE order
B:BOWER consultation
L:LIS consultation
R:Research
D:DEC system administration
P:Problems email
S:Sugar administration
r:Reading NetNews

Each entry is a Project that you can track your time working on. There is no project hierarchy - that is, you can't create sub-projects (or you will have to enter them as separate projects). This isn't an issue for me, personally.

Now you can open a terminal and type "worklog" to start Worklog. You'll see something like this:

So if you want to start logging time on your BOWER consultation, type "B" and the timer will start. Type "R" for tracking "Research" time, or "r" for "Reading NetNews". Note the bottom entry in the projects list, "no category yet". Pressing Enter ("CR" stands for "Carriage Return", meaning "Enter") will start logging time automatically and you will have the option of assigning an existing category to that time later on.

Press "Delete" (and again when they tell you to) to quit the program.

Note that when you start Worklog for the first time, it creates a new text file in your home directory for EACH entry in the "projects" file:

Because of this, I prefer to use a subdirectory for Worklog. I create a folder called "worklog" with my "projects" file inside, then I run Worklog with

cd ~/worklog && worklog

so my home directory stays a little cleaner. I've actually created a bash alias for this command (plus an additional command to cd back to my home directory after I quit Worklog):

alias worktimer='cd ~/worklog && worklog && cd ~/'

[Update: I just learned a trick from this Lifehacker comment - create a text file in your home directory called ".hidden" that lists files and folders that should be, well, hidden by default. This is another good way to keep your home directory clean. It works like a charm.]

Here are a few more screenshots that show operation of Worklog:

I pressed "B" to track the Bower consultation time, then pressed "Space" to pause the clock. The instructions are pretty straightforward from here. Note: "31.00 seconds (total 31.00 seconds)" - it tracks the time in the current session as well as the total time you've spent working on the project. If you close Worklog and come back later, it'll still remember how much time you've spent on your project. You can use "+" and "-" to add or subtract any amount of time as well.

Press "Enter" to stop tracking time and enter a comment on your work in this session. Press "Enter" again to go back to the main Worklog interface.

A kind of funky annoyance is that Worklog counts hours and minutes in hundredths, so 19.30 minutes is actually somewhere near 19 minutes and 18 seconds. It IS handy for calculating based on an hourly rate, though.

A nice perk...

Since Worklog stores all its data in the text files I mentioned previously, you can see a full summary of your work by opening the correct file in your text editor. Here, you can view or edit times, dates, or descriptions of your work. Want to start over each month? Just make a folder for the past month and move your old time logs in there to archive them, and Worklog will generate new clean files next time you start it up. The text files could be edited or read through scripting or synced via Dropbox. I find it really flexible and cool.

In conclusion

Worklog is pretty easy to use and is extremely lightweight, but there's not much information available about it and screenshots are few and far between. Hopefully this post helps out a little.
