Access XAMPP from Other Devices

Wed, 06 Mar 2013

Posted In: web

Tagged: linuxcode

I've tweaked my web development environment over the last couple days so I can access my local Wordpress websites from other machines on my network. These may include:

  • Other computers running different operating systems / browsers
  • Phones or tablets
  • Virtualbox guest OS's running on my computer

How I'm doing this

  1. Find your local IP address. Mine is 10.1.10.100. You can find this using a command like ifconfig for Linux.
  2. Set up XAMPP, MAMP, WAMP, LAMP, or whatever on your computer. I should point out that I'm using XAMPP on Arch Linux, and I can't guarantee another setup would work like this. If you try this, please leave a comment with your results!
  3. Install Wordpress. I use a Wordpress multi-site ("network") installation to keep things a little more streamlined - when I start a new project, I don't always need to create a new database for it, etc. It all depends on the project, but for normal theme development, this setup works fine.
  4. As you install Wordpress, install it under your local IP address (http://10.1.10.100) instead of http://localhost/
  5. When Wordpress is up, you'll view it at (for example) http://10.1.10.100/wordpress. You can use this same address to access it from any other device on your network.

Drawbacks

The biggest drawback is that this setup requires your local IP address always to be the same. If you take your computer to another network, you'll probably be assigned a different IP address, and there's not much you can do about this. Your Wordpress install won't particularly work, especially if you try to get past the home page. And changing the site URL is not really a quick and easy task when you account for media URLs and permalink-related issues.

In my case, I settled on a "good enough" solution. The benefits of this setup, especially with using Modern IE in Virtualbox, clearly outweigh the drawbacks for me.

I went in to my office and checked which local IP had been assigned to my machine. It was the one I mentioned above - 10.1.10.100. So then I came home, configured my router to serve address in the 10.1.10.xxx range, and assigned my computer the 10.1.10.100 IP address. So I have a working development environment both at home and at the office, which is good enough for me.
