May 2013 Personal Development Report

Thu, 30 May 2013

Posted In: web

Tagged: newspersonal

Inspired by a recent episode (I wish I could remember exactly which one) of The East Wing Podcast, I'm going to try to periodically write reviews of what I've been up to - web work I've done, things I'm learning, and what's on the horizon. This first post will be pretty general and hopefully I can be a little more specific next time.

Web

I'm currently plugging away on several WordPress-based websites, as usual. I'm pretty happy with some of the work that's going to be going live soon and I'm in the middle of a freelance project that I'm pretty excited about. I love working with WordPress, but I'm understanding that it's not always the best solution and I'm looking into a few alternatives as well.

Design

I don't consider myself a natural designer, so I try hard to look at great design and figure out what they're doing. Unmatched Style is an interesting blog that showcases great design with some good commentary. I would prefer to go in a more minimalist design route, but ultimately in some cases the client wouldn't prefer that style (or it wouldn't be right for their website). Staying detail-oriented and not over-designing is the key.

Front End

I'm currently rocking SCSS for EVERY project. Preprocessing CSS makes everything easier and better. This has led me to Zurb's Foundation framework, which I'm using on my current projects. I like Foundation and it provides a lot of good starter styles and javascript plugins, but some of it is more than I need. In the interest of keeping things leaner and more manageable locally, I'm considering pruning things down quite a bit. Foundation is not seeming quite as modular as I'd really prefer, so I may build my own assets for a skeleton-style front-end framework, borrowing styles from here and there. The Codrops Blueprints elements look really interesting to me as well.

I want to do more work in Javascript. I do a lot of code snippets in jQuery as well as some more advanced AJAXy stuff, but if I'm interested in building more interactive / engaging interfaces, I'll need to brush up on my plain JS.

Back End

I've been realizing lately that I'm more frontend-y than I'd prefer. Time to get back into some back-end programming. While I do some PHP to add functionality to WordPress themes, I'd like to work on more standalone stuff. PHP is an obvious choice because it's widely used, but I'm intrigued by a couple other languages:

  • Python is a nice and clean, well-supported language. It has obvious uses both on desktop and web.
  • Node.js is pretty young, but can do some amazing stuff. Meteor.js looks like a great framework for Node. It's very interesting to be able to write Javascript for both client and server using the same API's, and the realtime possibilites are exciting.

I'll need to just choose something and do it. Probably working on Javascript would have the most payoff in the future because of the frontend and backend applications.

Personal

I am pretty out of shape, but was able to complete my first triathlon during this month. My main claim to fame is that I had the second best bike -> run transition in the race. My swim leg was surprisingly not awful. I'd like to get in shape for a multi-day bike tour in Central / SE Oregon.

I'm not sure if I'll have time to do that. My days are busy and I have to get work done when I can. A lot of interruptions come up.

That's where things are at the moment. Hopefully I'll have a better report next time around - stay tuned.
