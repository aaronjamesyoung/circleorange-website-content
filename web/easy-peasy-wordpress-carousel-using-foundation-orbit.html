Easy-peasy WordPress carousel using Foundation Orbit

Mon, 28 Oct 2013

Posted In: web

Tagged: wordpressfoundationcode

I'm using a fair amount of pieces from Zurb's Foundation framework lately. Here's a simple way to create a basic image slider in WordPress for your theme, using the Orbit script from Foundation.

Advanced Custom Fields

First, install Advanced Custom Fields on your WordPress site. This is not meant to be an ACF tutorial, but basically, this is what you do:

  • Install and activate the plugin
  • Go to the "Custom Fields" entry near the bottom of the left-hand menu in WordPress. Add a new field group and give it an appropriate name.
  • For this project, add five image fields which will output "Image ID". I usually make the first field mandatory and the others optional.
  • Create a rule for what page (or post type) you want to have the custom fields appear on. I generally use "page = front page".

Hook up Foundation

If you're already using Foundation on your project, that's great! Otherwise, head off to http://foundation.zurb.com/ and download a Foundation pack containing at least the Orbit CSS styles. Personally, I use the SCSS version of Foundation. In any case, make sure that your stylesheet contains the CSS styles for the Orbit slider.

Javascript

Also make sure that your theme is using the correct Javascript files for Orbit. You'll need the basic Foundation javascript file as well as the Orbit-specific javascript. Lastly, see the Foundation Orbit documentation for some tips and tricks as well as the javascript setup to make your slider work. See the "Using the Javascript" section.

OK, putting all this together in your WordPress theme

Now for the meat and potatoes. I've created a Gist with this code to refer to:

Orbit Slider Github Gist

In your theme - perhaps in your front-page.php file - is where that code goes. Here's a breakdown:

<?php $image1 = wp_get_attachment_image_src( get_field('slider_1'), 'home-slider-size' );
  $image2 = wp_get_attachment_image_src( get_field('slider_2'), 'home-slider-size' );
  $image3 = wp_get_attachment_image_src( get_field('slider_3'), 'home-slider-size' );
  $image4 = wp_get_attachment_image_src( get_field('slider_4'), 'home-slider-size' );
  $image5 = wp_get_attachment_image_src( get_field('slider_5'), 'home-slider-size' );
?>

This sets up the variables for each image URL that you've entered in Advanced Custom Fields. Each variable in this case contains an array, the first element of which is the URL for the image. So in order to use these variables, you'll need to write, e.g., echo $image1[0];.

I've also used my functions.php file to define a custom image size for the slider images, called "home-slider-size".

Then we'll write the markup for the Orbit slider:

<ul data-orbit>
  <li>
    <img src="<?php echo $image1[0]; ?>" />
  </li>
  <?php if(get_field('slider_2')) { ?>
  <li>
    <img src="<?php echo $image2[0]; ?>" />
  </li>
  <?php } ?>
  ... and so on ...
</ul>

This way, we have a mandatory first image for the slider (which will just sit there if there are no other images). Subsequent images are optional - so if you don't need to use five different slides at one time, that's OK.

Wrapping up

This is a nice, easy way for clients to manage the slider (if they need one) on the home page of their website. I often have optional captions per slide as well via Advanced Custom Fields.

Personally, I often find that many of the prepackaged WordPress carousel plugins are either overkill, too simple, not customizable enough, or too difficult for clients to use easily. This way they can just go to edit their home page to update the carousel, which is generally pretty intuitive.
